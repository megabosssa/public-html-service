Istio Virtual Service Generator

Generate Istio VirtualService direct response YAML for mocking API endpoints

What is this?
Generate mock HTTP responses directly from Istio without needing a backend service. Perfect for testing and development.
Where to place?
Add the generated YAML under spec.http in your VirtualService resource file.
Pro Tip
Use Ctrl+Enter to quickly add a mock. Order matters - first match wins in Istio!

Usage Example

Place the generated YAML inside your VirtualService:

apiVersion: networking.istio.io/v1beta1
kind: VirtualService
metadata:
  name: my-service-vs
spec:
  hosts:
    - my-service
  http:
    # ↓ Paste generated mocks here ↓
    - directResponse:
        body:
          string: |
            {"status": "ok"}
        status: 200
      match:
        - uri:
            exact: /health
    # ↓ Your actual routes below ↓
    - route:
        - destination:
            host: my-service
